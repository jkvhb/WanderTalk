318路书

路书规划 & 动画讲解视频生成工具 · 产品需求文档 v1.0

一、产品定位

318路书 是一个面向自驾旅行者的路书规划与讲解视频生成工具。


用户在地图上自由规划多日自驾路线、搜索并添加沿途的餐饮住宿等兴趣点，完成路书后一键生成 飞行动画 + AI旁白讲解 的路书视频，像纪录片一样展示整条路线。

一句话描述

在地图上规划自驾路线，自动生成带旁白讲解的飞行动画视频。

核心差异化

对比维度路小绘 TrailPaint318路书（本产品）
POI 数据搜索能力有限，很多地点搜不到基于高德地图，覆盖餐饮/住宿/景点等全类别
路线规划基础路线绘制多日分段规划 + 沿实际道路的驾车路线 + 距离/时间计算
分享形式静态路书图片/页面动画讲解视频（飞行动画 + 旁白配音）
语音播报无AI 自然语音旁白，支持停顿、语气，效果接近真人讲解

三、功能全景

F1 · 地图交互

  • 高德地图底图，支持标准/卫星图层切换
  • 缩放、平移、定位
  • 点击地图任意位置查看坐标

F2 · POI 搜索

  • 关键词搜索地点、餐厅、酒店、景点
  • 分类筛选：🍜 餐饮 🏠 住宿 📍 景点 ⛽ 加油站 📸 观景台
  • 搜索结果展示名称、地址、评分、人均消费
  • 一键添加到路线中

F3 · 路线规划（核心）

  • 完全自由的路线编辑——不限于318，任何路线都可以
  • 预设 318 川藏线 19 个关键节点作为快速模板，但每个节点都可以增删改
  • 多天分段规划：每天包含起点→途经点→终点
  • 实际驾车路线计算（沿真实道路，非直线）
  • 显示每段距离、预计时间
  • 支持添加途经点（多段拼接）
  • 每个节点可编辑：名称、到达时间、备注

F4 · 旁白编辑

  • 为每段路线编写讲解文案
  • 支持 SSML 标记（控制停顿、语速、重音），实现自然播报效果
  • 多种 AI 语音可选：晓晓（活泼女声） 云希（稳重男声） 晓伊（温柔女声）
  • 一键合成试听，不满意可反复调整

F5 · 动画预览

  • 在地图上以飞行动画形式预览整条路线
  • 相机自动跟随路线移动，支持暂停/拖拽进度
  • 旁白与画面自动同步——旁白长就飞慢点，旁白短就飞快点
  • 途经重要节点时自动叠加地名、海拔标签
  • 到达关键站点时自动停留，等旁白说完再继续

F6 · 视频导出

  • 将动画+旁白导出为视频文件（WebM 格式）
  • 支持 720p / 1080p 分辨率
  • 导出时显示进度条
  • 导出完成后自动下载

F7 · 数据持久化

  • 路线数据自动保存到浏览器（IndexedDB）
  • 关闭浏览器再打开，之前的路线还在
  • 旁白音频缓存，避免重复合成

六、核心算法：旁白驱动动画

这是本产品最关键的技术设计——保证动画节奏始终跟随旁白，绝不出现"话没说完画面就跑了"。

原理

不是实时同步两个独立系统，而是先知道旁白有多长，再决定动画飞多快。整个过程是确定性计算，不涉及 AI 预测。

三步流程

第一步：合成旁白 → 拿到精确时长 段1 "从成都出发，沿成雅高速一路向西..." → TTS → 14.2秒 段2 "翻越折多山垭口，海拔4298米..." → TTS → 8.7秒 段3 "抵达摄影天堂新都桥..." → TTS → 11.5秒 第二步：用旁白时长反推动画节奏 段1: 距离85km / 200kmh虚拟速度 = 至少需要5秒飞完 旁白14.2秒 > 飞行5秒 → 剩余9.2秒分配为停留缓冲 → 起点停留2秒 + 飞行放缓至7秒 + 终点停留5.2秒 = 14.2秒 ✓ 第三步：生成关键帧时间轴 0s 2s 9s 14.2s ├────────┼───────────────┼─────────────┤ 停留成都 飞行动画→→→→→→ 到达康定停留 旁白开始→→→→→→→→→→→→→→→→→→→→旁白结束 "雅安"标签渐入/渐出

两种场景自动处理

场景策略
旁白长、路程短在起终点自动插入停留时间，相机原地悬停展示场景
旁白短、路程长加快飞行速度，或在途经点之间自然过渡
段与段之间自动插入 1-2 秒过渡，画面保持 + 旁白自然停顿

旁白自然停顿（SSML）

<speak> 现在我们离开成都，<break time="300ms"/> 沿成雅高速向雅安方向前进。<break time="500ms"/> 二郎山隧道<emphasis>是川藏线上的第一道门户</emphasis>， <break time="200ms"/> 穿过隧道，就正式进入甘孜藏区了。 </speak>

通过 <break> 标签控制停顿，通过 <emphasis> 控制重音，让播报有语气有节奏。

七、技术架构

双地图引擎

为什么需要两个地图库？因为它们的用途不同：

阶段地图库原因
路线编辑高德 JS API 2.0国内 POI 数据最全，餐饮/住宿都能搜到；原生驾车路线规划
视频导出MapLibre GL JSCanvas 可控，支持 captureStream() 录制；用高德免费瓦片，不消耗 API 额度

技术选型理由

技术用途为什么选它
Vue 3 + Vite前端框架响应式数据绑定适合地图交互；Vite 开发快、构建产物小
Pinia状态管理路线数据在多个组件间共享，Pinia 是 Vue 3 官方推荐
高德 JS API 2.0地图 + POI + 路线国内唯一靠谱选择，POI 覆盖最全，免费额度够用
Edge TTS语音合成微软神经网络语音，zh-CN-XiaoxiaoNeural 效果自然；免费、无需后端、无需 API Key
MapLibre GL JS视频导出渲染开源、Canvas 可控、支持 captureStream(30fps)
IndexedDB (idb)本地存储路线坐标数据量大(几千个点)，localStorage 只有 5MB 不够用
MediaRecorder视频编码浏览器原生 API，Canvas captureStream + 音频混合 → WebM

无后端设计

整个应用零后端依赖：

  • 地图瓦片：高德免费 CDN
  • TTS 语音：Edge TTS 免费接口（浏览器直连微软服务器）
  • 路线计算：高德 API（前端直接调用）
  • 数据存储：浏览器 IndexedDB
  • 视频编码：浏览器 MediaRecorder
  • 部署产物：纯静态 HTML + JS + CSS

唯一外部依赖是用户自己的高德 API Key（免费申请，5分钟搞定）。

十一、风险与应对

风险等级应对
Edge TTS 请求频率限制中顺序合成 + 请求间隔 2 秒 + 结果缓存 IndexedDB，后续导出直接复用
Edge TTS 未来收费/限制浏览器低架构预留切换其他 TTS 的能力；备选：浏览器 Web Speech API（质量降级但可用）
高德 API 日调用量超限低POI 搜索缓存；路线数据一次计算后存入 IndexedDB 复用
视频导出时浏览器内存溢出中默认 720p；视频时长限制 5 分钟；分段渲染可后期优化
iOS Safari 不支持 MediaRecorder高检测兼容性并提示用户换 Chrome；长期可考虑服务端渲染方案
captureStream 在部分 GPU 上黑屏中降级方案：逐帧 requestAnimationFrame + Canvas drawImage 手动录制
