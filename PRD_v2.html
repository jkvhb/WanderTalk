一、产品定位

318路书 是一个面向自驾旅行者的路书规划与讲解视频生成工具。


用户在地图上自由规划多日自驾路线、搜索并添加沿途的餐饮住宿等兴趣点，完成路书后一键生成 飞行动画 + AI旁白讲解 的路书视频，像纪录片一样展示整条路线。

一句话描述

在地图上规划自驾路线，自动生成带旁白讲解的飞行动画视频。

核心差异化

对比维度路小绘 TrailPaint318路书（本产品）
POI 数据搜索能力有限基于高德地图，覆盖餐饮/住宿/景点等全类别
路线规划基础路线绘制多日分段规划 + 沿实际道路驾车路线 + 距离/时间
分享形式静态路书图片/页面动画讲解视频（飞行动画 + 旁白配音）
旁白生产无AI 辅助生成旁白草稿 + 自然语音合成

两个阶段目标

个人版（当前目标）
自用工具，纯前端，无后端，用户自备高德 API Key。功能完整但不考虑多用户场景。
公开版（未来扩展）
加轻量后端代理，统一管理 API Key 和 TTS 服务。核心代码无需重写，迁移成本约 1~2 周。

三、功能全景

F1 · 地图交互

  • 高德地图底图，支持标准/卫星图层切换
  • 缩放、平移、定位
  • 点击地图任意位置添加自定义节点

F2 · POI 搜索

  • 关键词搜索地点、餐厅、酒店、景点
  • 分类筛选：🍜 餐饮 🏠 住宿 📍 景点 ⛽ 加油站 📸 观景台
  • 搜索结果展示名称、地址、评分、人均消费
  • 一键添加到路线中
  • 搜索结果缓存，避免重复调用 API

F3 · 路线规划（核心）

  • 完全自由的路线编辑——不限于318，任何路线都可以
  • 预设 318 川藏线 9 天关键节点作为快速模板，每个节点都可增删改
  • 多天分段规划：每天包含起点→途经点→终点
  • 实际驾车路线计算（沿真实道路，非直线），结果缓存到 IndexedDB
  • 显示每段距离、预计时间
  • 每个节点可编辑：名称、备注、到达时间
  • 路书导出为 JSON 文件 / 从 JSON 文件导入（防止清缓存丢失数据）

F4 · 旁白编辑

  • 为每段路线编写讲解文案
  • AI 辅助生成旁白草稿：根据路线节点名称、距离、海拔等信息自动生成初稿，用户在此基础上修改
  • 支持 SSML 标记（控制停顿、语速、重音），实现自然播报效果
  • 多种语音可选：晓晓（活泼女声） 云希（稳重男声） 晓伊（温柔女声）
  • 一键合成试听，不满意可反复调整；合成结果缓存，避免重复请求

F5 · 动画预览

  • 在地图上以飞行动画形式预览整条路线
  • 相机自动跟随路线移动，支持播放/暂停/拖拽进度
  • 旁白与画面自动同步——先拿到旁白时长，再反推动画节奏
  • 途经重要节点时自动叠加地名、海拔标签（渐入渐出）
  • 到达关键站点时自动停留，等旁白说完再继续

F6 · 视频导出

  • 将动画+旁白导出为视频文件（WebM 格式）
  • 支持 720p（默认）/ 1080p 分辨率
  • 导出时显示进度条，完成后自动下载
  • 导出前检测浏览器兼容性，不兼容时给出明确提示（需 Chrome 88+）
  • 提供 WebM → MP4 转换建议（ffmpeg 命令 / 在线工具链接）

F7 · 数据持久化

  • 路线数据自动保存到 IndexedDB
  • 旁白音频 Blob 缓存，避免重复合成
  • 路书 JSON 导出/导入，换浏览器/换设备不丢数据

六、核心算法：旁白驱动动画

这是本产品最关键的技术设计——保证动画节奏始终跟随旁白，绝不出现"话没说完画面就跑了"。

原理

不是实时同步两个独立系统，而是先知道旁白有多长，再决定动画飞多快。整个过程是确定性计算，不涉及 AI 预测。

三步流程

第一步：合成旁白 → 拿到精确时长 段1 "从成都出发，沿成雅高速一路向西..." → TTS → 14.2秒 段2 "翻越折多山垭口，海拔4298米..." → TTS → 8.7秒 段3 "抵达摄影天堂新都桥..." → TTS → 11.5秒 第二步：用旁白时长反推动画节奏 段1: 距离85km / 200kmh虚拟速度 = 至少需要5秒飞完 旁白14.2秒 > 飞行5秒 → 剩余9.2秒分配为停留缓冲 → 起点停留2秒 + 飞行放缓至7秒 + 终点停留5.2秒 = 14.2秒 ✓ 第三步：生成关键帧时间轴 0s 2s 9s 14.2s ├────────┼───────────────┼─────────────┤ 停留成都 飞行动画→→→→→→ 到达康定停留 旁白开始→→→→→→→→→→→→→→→→→→→→旁白结束 "雅安"标签渐入/渐出

两种场景自动处理

场景策略
旁白长、路程短在起终点自动插入停留时间，相机原地悬停展示场景
旁白短、路程长加快飞行速度，或在途经点之间自然过渡
段与段之间自动插入 1~2 秒过渡，画面保持 + 旁白自然停顿

SSML 旁白示例

<speak> 现在我们离开成都，<break time="300ms"/> 沿成雅高速向雅安方向前进。<break time="500ms"/> 二郎山隧道<emphasis>是川藏线上的第一道门户</emphasis>， <break time="200ms"/> 穿过隧道，就正式进入甘孜藏区了。 </speak>

七、技术架构

双地图引擎与坐标系处理

本产品使用两套地图库，它们的坐标系不同，必须在数据层统一转换：

阶段地图库坐标系说明
路线编辑高德 JS API 2.0GCJ-02（火星坐标）POI 搜索、驾车路线规划，返回的坐标均为 GCJ-02
视频导出MapLibre GL JSWGS-84（标准经纬度）开源渲染，Canvas 可录制
坐标系偏移问题（必须处理）
GCJ-02 与 WGS-84 之间存在 50~500 米偏差，在城区尤为明显。
解决方案：所有从高德获取的坐标在存入数据模型前，统一通过 gcj02ToWgs84(lng, lat) 转换为 WGS-84 存储；高德地图渲染时反向转换回 GCJ-02 用于显示，MapLibre 直接使用存储的 WGS-84 坐标。
坐标处理流： 高德 API 返回 GCJ-02 坐标 ↓ gcj02ToWgs84() 数据模型统一存储 WGS-84 ↓ ↓ wgs84ToGcj02() 直接使用 高德地图显示 MapLibre 渲染/录制

八、AI 旁白辅助生成

旁白文案是用户最大的使用门槛。系统根据路线数据自动生成旁白草稿，降低从零开始写作的难度。

草稿生成逻辑

调用大语言模型 API（如 Claude / 通义千问），传入以下结构化信息，输出带 SSML 标记的旁白草稿：

输入： { "segment": "Day 2 · 康定 → 雅江", "distance": "176km", "duration": "约5小时", "keyPoints": [ { "name": "折多山垭口", "altitude": 4298, "note": "川藏线第一高垭口" }, { "name": "新都桥", "note": "摄影天堂，光线极佳" } ] } 输出（示例）： 告别康定，今天我们迎来川藏线上的第一个挑战。 <break time="400ms"/> 折多山垭口，海拔<emphasis>4298米</emphasis>， 是进入高原的标志性关口...

个人版实现方式

个人版中，用户在设置页填入自己的 Claude API Key 或通义千问 API Key，旁白生成请求直接从浏览器发出，无需后端。与高德 API Key 的管理方式一致。

旁白编辑交互

  1. 用户进入视频工作室，每段路线下方显示旁白编辑框
  2. 点击"AI 生成草稿"按钮，2~5 秒内返回初稿
  3. 用户在文本框内自由修改，支持插入 SSML 标签
  4. 点击"试听"按钮合成并播放，不满意继续修改
  5. 所有段落确认后，点击"全部合成"批量生成音频并缓存

十二、风险与应对

风险等级应对方案
Edge TTS 服务中断 高 TTS 适配器封装隔离，中断时一键切换为浏览器 Web Speech API（质量降级但不崩溃）；公开版改为火山引擎
iOS Safari 不支持 MediaRecorder 高 加载时检测并展示提示横幅；路书规划和预览功能不受影响；长期可追加服务端渲染方案
坐标系偏移导致路线漂移 高 Phase 2 第一个任务即建立坐标转换层，数据模型统一存 WGS-84，从根本上消除此问题
视频导出内存溢出 中 默认 720p；视频时长建议 ≤10 分钟；分段渲染作为后期优化项
captureStream 在部分 GPU 上黑屏 中 降级为 rAF 逐帧 Canvas 录制（Phase 5 必须实现此降级路径）
高德 API 日调用量超限 低 POI 搜索结果缓存；路线计算结果存 IndexedDB 复用；个人使用几乎不会超限
LLM API 费用（旁白生成） 低 个人版用户自备 Key 自付费用；生成一次完整路书旁白约 2000~4000 tokens，成本 ¥0.01 以内
